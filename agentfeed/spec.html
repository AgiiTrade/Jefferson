📄 AgentFeed Specification v1.0

The open standard for publishing content that AI agents can read, interpret, and relay.

Overview

AgentFeed is a JSON-based feed format designed for AI agent consumption. Unlike RSS (designed for human readers) or API responses (designed for applications), AgentFeed is structured specifically for AI agents to parse, filter, and relay to their human operators.

Feed Structure

{
  "feedVersion": "1.0",
  "publisher": "string required",
  "publisherUrl": "string optional",
  "feedUrl": "string optional",
  "lastUpdated": "ISO-8601 required",
  "language": "ISO-639-1 optional (default: en)",
  "topics": ["string"] optional,
  "items": [Item] required
}

Feed Fields

FieldTypeDescription
feedVersionstringFormat version. Current: "1.0"
publisherstringCompany or individual name
publisherUrlstringURL to publisher's website
feedUrlstringCanonical URL of this feed
lastUpdatedISO-8601When the feed was last modified
languagestringISO-639-1 language code (en, fr, es, hi, ur, etc.)
topicsstring[]Global topics for the entire feed
itemsItem[]Array of feed items (see below)

Item Structure

{
  "id": "string required",
  "type": "string required",
  "headline": "string required",
  "summary": "string required",
  "keyTakeaway": "string required",
  "tags": ["string"] optional,
  "urgency": "string optional",
  "audience": ["string"] optional",
  "publishedAt": "ISO-8601 required",
  "expiresAt": "ISO-8601 optional",
  "cta": {
    "text": "string",
    "url": "string"
  } optional,
  "source": "string optional",
  "image": "string URL optional"
}

Item Fields

FieldTypeDescription
idstringUnique identifier within the feed
typestringContent type (see below)
headlinestringWhat happened — concise, factual
summarystringWhy it matters — 2-3 sentences
keyTakeawaystringThe ONE thing the agent should relay
tagsstring[]Topic tags for filtering
urgencystringlow, medium, or high
audiencestring[]Target roles (cto, cfo, ceo, marketing, etc.)
publishedAtISO-8601When the item was published
expiresAtISO-8601When the item should no longer be surfaced
ctaobjectOptional call-to-action with text and URL
sourcestringSource URL or reference
imagestringOptional image URL

Agent Behavior Guidelines

When an AI agent processes an AgentFeed, it should:

  1. Filter by relevance — Match tags and audience against the operator's interests and role.
  2. Respect urgency — Surface high urgency items immediately. Batch medium into daily digests. Surface low only when contextually relevant.
  3. Check expiration — Never surface items past their expiresAt date.
  4. Relay the takeaway — The keyTakeaway field is designed to be the agent's primary message to the operator. Summarize further if needed.
  5. Preserve attribution — Always credit the publisher when relaying content.

Discovery

Publishers can make their feeds discoverable by adding a meta tag to their website:

<link rel="agentfeed" type="application/json" href="https://yourdomain.com/agentfeed.json">

Agents can discover feeds by checking for this tag, similar to how RSS autodiscovery works.

Versioning

The feedVersion field indicates the spec version. Future versions will be backward-compatible — new fields added, existing fields preserved. Breaking changes will increment the major version.
